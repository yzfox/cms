# 人口普查管理系统项目总结报告

## 1. 项目概述

人口普查管理系统（CMS）是一个基于Spring Boot + Vue 3开发的现代化前后端分离应用，专为高效管理和统计人口普查数据而设计。系统包含网页管理后台和微信小程序两个客户端，提供了完整的户籍、居民、教育、医疗、就业、社保、房产、车辆等信息的管理功能，并支持多维度的数据统计和可视化展示。

本项目旨在解决传统人口普查数据管理效率低下、数据不统一、统计分析困难等问题，通过现代化的技术架构和友好的用户界面，显著提升人口普查工作的效率和质量。

### 2.2 技术栈详解

## 3. 核心功能模块

### 3.1 业务模块
1. **户籍管理**: 完整的户籍信息CRUD操作，支持按户籍编号和户主姓名搜索
2. **居民管理**: 居民信息管理与户籍关联，支持按姓名和身份证号搜索
3. **教育管理**: 教育背景信息管理，与居民信息关联确保数据一致性
4. **医疗管理**: 医疗保障信息管理，支持按居民姓名和医院名称搜索
5. **就业管理**: 就业情况信息管理，支持按居民姓名和工作单位搜索
6. **社保管理**: 社会保险信息管理，支持按居民姓名和参保状态搜索
7. **房产管理**: 房产信息管理，支持按多种条件搜索，使用BigDecimal处理金额和面积
8. **车辆管理**: 车辆信息管理，支持按多种条件搜索，使用BigDecimal处理购置价格

### 3.2 系统功能
1. **用户认证**: 基于JWT的登录认证，使用BCrypt加密算法处理密码
2. **权限控制**: 基于角色的访问控制（RBAC），动态菜单和路由生成
3. **统计分析**: 多维度数据统计，包括地区分布、人口增长趋势、年龄分布等
4. **数据可视化**: ECharts图表展示，支持柱状图、折线图、饼图、地图等多种图表类型
5. **搜索筛选**: 支持各模块的高级搜索和筛选功能
6. **分页处理**: 所有列表支持分页，提高大数据量下的性能
7. **实时通信**: WebSocket数据实时更新，确保数据一致性
8. **缓存管理**: 精细化的缓存管理，针对统计类高频查询结果进行缓存与即时清除

## 4. 安全机制

1. **SQL注入防护**: 使用MyBatis Plus的参数化查询，避免直接拼接SQL语句
2. **XSS攻击防护**: 前后端双重防护，前端输入验证和转义，后端对输入数据进行过滤
3. **CSRF防护**: 使用JWT Token机制，无状态认证避免CSRF攻击
4. **接口访问控制**: 基于JWT Token的接口保护，权限验证确保只有授权用户才能访问特定接口
5. **密码安全**: 使用BCrypt加密算法，密码加盐处理提高安全性

## 6. 部署与配置

### 6.1 环境要求
- JDK 17+
- MySQL 8.0+
- Node.js 14+
- Maven 3.6+
- 微信开发者工具（用于小程序开发）

### 6.2 部署步骤
1. 克隆项目并进入项目目录
2. 初始化数据库，执行 `database_cms.sql` 脚本
3. 修改后端配置文件 `src/main/resources/application.properties`
4. 修改前端配置文件 `vue/src/utils/request.js`
5. 修改小程序配置文件 `cms-miniprogram/utils/request.js`
6. 启动后端服务：`mvn spring-boot:run`
7. 启动前端服务：`cd vue && npm install && npm run serve`
8. 使用微信开发者工具打开小程序项目

### 6.3 IP配置说明
当电脑的网络IP地址变化时，可能会出现跨域访问错误。解决方案：
1. 临时方案：更新后端CORS配置中的allowedOrigins列表
2. 永久方案：将allowedOrigins替换为allowedOriginPatterns，允许所有IP访问
