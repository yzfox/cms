# CMS Android项目开发流程文档

## 1. 项目概述

CMS Android项目是CMS内容管理系统的一个移动端客户端，旨在为普通用户提供便捷的移动访问体验。该项目基于Java语言开发，采用原生Android开发方式，与后端Spring Boot服务进行数据交互。

## 2. 技术栈

- **开发语言**: Java
- **网络请求库**: OkHttp
- **JSON解析库**: Gson
- **UI框架**: Material Design Components
- **架构模式**: MVP（Model-View-Presenter）
- **数据存储**: SharedPreferences（Token存储）

## 5. 开发规范

### 5.1 代码规范
- 遵循Google Java Style Guide
- 使用统一的命名规范
- 合理使用注释和文档
- 保持代码简洁和可读性

### 5.2 Git版本控制
- 主分支：`main`（稳定版本）
- 开发分支：`develop`（日常开发）
- 特性分支：`feature/xxx`（新功能开发）
- 修复分支：`fix/xxx`（bug修复）
- 发布分支：`release/xxx`（版本发布）

### 5.3 提交规范
- 提交信息清晰明了
- 每个提交只包含一个功能或修复
- 使用语义化提交信息（如feat: 添加登录功能）
