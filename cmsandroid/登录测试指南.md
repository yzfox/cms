# CMS Android登录功能测试指南

## 1. 测试前准备

### 1.1 确保后端服务已启动
- 后端服务默认运行在 `http://localhost:8080`
- 确保MySQL数据库已启动并正确配置
- 确保数据库中已导入初始数据

### 1.2 已知测试账户
根据数据库脚本，系统中预置了以下用户：

| 用户名 | 密码 | 角色 | 说明 |
|--------|------|------|------|
| admin | 123456 | ADMIN | 管理员账户 |
| user | 123456 | USER | 普通用户账户 |

## 2. 登录功能测试步骤

### 2.1 启动Android应用
1. 在Android Studio中打开cms-android项目
2. 确保设备或模拟器已连接
3. 点击"Run"按钮启动应用

### 2.2 执行登录测试
1. 应用启动后会自动跳转到登录页面
2. 输入用户名和密码：
   - 管理员登录：用户名 `admin`，密码 `123456`
   - 普通用户登录：用户名 `user`，密码 `123456`
3. 点击"登录"按钮
4. 观察登录结果

### 2.3 预期结果
- 登录成功后，应用会保存JWT Token并跳转到主页面
- 主页面应显示"居民管理"和"退出登录"按钮
- 点击"居民管理"应能进入居民列表页面
- 点击"退出登录"应清除Token并返回登录页面

## 3. 登录功能验证点

### 3.1 成功登录验证
- [ ] 正确的用户名和密码能够成功登录
- [ ] 登录成功后Token被正确保存
- [ ] 登录成功后页面正确跳转到主页面
- [ ] 主页面功能按钮正常显示

### 3.2 异常情况验证
- [ ] 错误的用户名或密码应提示相应错误信息
- [ ] 空用户名或密码应提示输入验证信息
- [ ] 网络异常时应提示网络错误信息
- [ ] Token过期或无效时应自动跳转回登录页面

## 4. 常见问题排查

### 4.1 网络连接问题
- 确认后端服务已在运行
- 检查模拟器或设备网络连接是否正常
- 确认API地址配置是否正确（默认为10.0.2.2:8080）

### 4.2 数据库问题
- 确认MySQL服务已启动
- 确认数据库中已导入初始数据
- 检查数据库连接配置是否正确

### 4.3 权限问题
- 确认AndroidManifest.xml中已添加网络权限
- 确认应用具有网络访问权限
